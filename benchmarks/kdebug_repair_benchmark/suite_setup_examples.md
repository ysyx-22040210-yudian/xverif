# Suite Setup Examples v2

These examples show the expected case-local scripts.  They are templates, not
private answers.

## Evidence Setup Required for Every Case

Every case must be failed first and then receive its own collection plan. Do
not prepopulate `evidence/with_kdebug/` by copying files:

```text
case_001/
  fail/run.log
  inputs/waves.fsdb
  evidence/
    kdebug_plan.json
    requests/failure_value.json
    with_kdebug/                 # generated by kdebug_evidence.py
```

Start from these mutually consistent templates:

```text
templates/case_meta.template.json
templates/kdebug_evidence_plan.template.json
templates/kdebug_request.value_at.template.json
```

Replace the FSDB/daidir, signal, action, and time with real current-case values.
Then run, as ordinary user `host`:

```bash
/usr/local/bin/python3.8 /home/host/kverif/benchmarks/kdebug_repair_benchmark/scripts/kdebug_evidence.py \
  collect --case-dir /home/host/kverif_runs/suite/case_001 \
  --kdebug /home/host/kverif/tools/kdebug
```

See [`benchmark_evidence_gate.md`](benchmark_evidence_gate.md) for the complete
manifest and rejection contract.

## Generated RTL UVM Case

Example repair workdir:

```text
case_001/
  case_meta.json
  rtl/
  tb/
  cases/
  Makefile
  scripts/
    build.sh
    run.sh
    judge.sh
  evidence/
    kdebug_plan.json
    requests/failure_value.json
    with_kdebug/
```

`case_meta.json` for an RTL-only generated case:

```json
{
  "bug_domain": "rtl",
  "public_design_refs": [
    "design_refs/common/rtl_debug_common.md",
    "design_refs/ut/axi/axi_ut_design.md"
  ],
  "repair_scope": {
    "allowed_roots": ["rtl"],
    "requires_rtl_change": true,
    "requires_env_change": false,
    "forbidden_files": ["answer_key_private.json", "scripts/judge.sh"]
  },
  "tool_evidence": {
    "required_for_with_kdebug": true,
    "plan": "evidence/kdebug_plan.json",
    "expected_files": ["kdebug_failure_value.json"],
    "minimum_nonempty_files": 1
  }
}
```

`scripts/build.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail
source /opt/rh/devtoolset-11/enable
export VCS_HOME=/home/synopsys/vcs/O-2018.09-SP2
export VERDI_HOME=/home/synopsys/verdi/Verdi_O-2018.09-SP2
export VCS_TARGET_ARCH=linux64
export PATH="$VCS_HOME/bin:$VERDI_HOME/bin:$PATH"
export LM_LICENSE_FILE=27000@IC_EDA
export SNPSLMD_LICENSE_FILE=27000@IC_EDA
make compile BACKEND=generated
```

`scripts/run.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail
mkdir -p run
make run BACKEND=generated CASE="${BENCH_CASE:-ut_axi_burst_outstanding}" \
  STRESS_ITERS="${STRESS_ITERS:-200}" 2>&1 | tee run/run.log
```

`scripts/judge.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail
BENCH_ROOT="${BENCH_ROOT:-/home/host/kverif/benchmarks/kdebug_repair_benchmark}"
"$BENCH_ROOT/scripts/judge_uvm_generated.sh" run/run.log "${BENCH_CASE:-ut_axi_burst_outstanding}"
```

## Environment-Only Case

Example repair workdir:

```text
case_011/
  case_meta.json
  rtl/                  # visible for context but not required to change
  scripts/
    build.sh
    run.sh              # repairable
    judge.sh            # protected
  env/
  config/
  evidence/
    kdebug_plan.json
    requests/runtime_observation.json
    with_kdebug/
```

`case_meta.json`:

```json
{
  "bug_domain": "env",
  "public_design_refs": [
    "design_refs/common/rtl_debug_common.md",
    "design_refs/fullchip/memory_subsystem/memory_subsystem_fullchip_design.md"
  ],
  "repair_scope": {
    "allowed_roots": ["scripts", "env", "config", "filelists"],
    "allowed_files": ["Makefile", "filelist.f", "vcs_args.f", "run_args.txt"],
    "requires_rtl_change": false,
    "requires_env_change": true,
    "forbidden_files": ["answer_key_private.json", "scripts/judge.sh", "case_meta.json"]
  },
  "tool_evidence": {
    "required_for_with_kdebug": true,
    "plan": "evidence/kdebug_plan.json",
    "expected_files": ["kdebug_runtime_observation.json"],
    "minimum_nonempty_files": 1
  }
}
```

Environment bugs should affect the real build/run path, for example dropped
`+diff`, wrong `REF_SO`, stale `simv`, wrong run case, missing executable bit,
or too-short workload timeout.  Do not hide the problem in `scripts/judge.sh`;
the judge is protected.

## Mixed RTL + Environment Case

Example `case_meta.json`:

```json
{
  "bug_domain": "mixed",
  "public_design_refs": [
    "design_refs/common/rtl_debug_common.md",
    "design_refs/it/memory/memory_it_design.md"
  ],
  "repair_scope": {
    "allowed_roots": ["rtl", "scripts", "env", "config", "filelists"],
    "allowed_files": ["Makefile", "filelist.f", "vcs_args.f", "run_args.txt"],
    "requires_rtl_change": true,
    "requires_env_change": true,
    "forbidden_files": ["answer_key_private.json", "scripts/judge.sh", "case_meta.json"]
  },
  "tool_evidence": {
    "required_for_with_kdebug": true,
    "plan": "evidence/kdebug_plan.json",
    "expected_files": ["kdebug_rtl_observation.json", "kdebug_runtime_observation.json"],
    "minimum_nonempty_files": 1
  }
}
```

Mixed cases should require two real fixes.  Good examples:

- RTL memory mask/data beat error plus run script points at the wrong log path.
- RTL branch redirect error plus wrong `REF_SO` or missing `+diff`.
- RTL LSU/cache error plus too-short timeout or wrong workload argument.

## Full-Chip VCS Case

Example repair workdir:

```text
case_006/
  case_meta.json
  inputs/
    waves.fsdb
    simv.daidir/
  evidence/
    kdebug_plan.json
    requests/
    with_kdebug/
  rtl/
  scripts/
    build.sh
    run.sh
    judge.sh
```

`scripts/build.sh` should build from repair-local RTL.  In previous campaigns
this was done by a case-local `build_kdb.sh` or VCS command that points at
`./rtl`.

`scripts/run.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail
mkdir -p run
RUN_BIN="${RUN_BIN:-/root/XiangShan-build/ready-to-run/flash_recursion_test.bin}"
REF_SO="${REF_SO:-/root/XiangShan-build/ready-to-run/riscv64-nemu-interpreter-so}"
timeout "${RUN_TIMEOUT_SEC:-3300}" ./simv +workload="$RUN_BIN" +diff="$REF_SO" \
  -assert finish_maxfail=30 -assert global_finish_maxfail=10000 \
  2> run/assert.log | tee run/run.log
```

`scripts/judge.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail
BENCH_ROOT="${BENCH_ROOT:-/home/host/kverif/benchmarks/kdebug_repair_benchmark}"
"$BENCH_ROOT/scripts/judge_fullchip.sh" run/run.log "${REQUIRE_MICROBENCH_PASS:-0}"
```

## Trial Launch

The supported entry point is `run_matrix.sh`, because it enforces collection
and suite validation before trial scheduling:

```bash
su - host

export KDEBUG_BIN=/home/host/kverif/tools/kdebug
export KDEBUG_PYTHON=/usr/local/bin/python3.8
export KDEBUG_REPORT_PYTHON=/bin/python3
export MAAS_BASE_URL=https://maas-coding-api.cn-huabei-1.xf-yun.com/v2
export MAAS_API_KEY=<provided key>

bash /home/host/kverif/benchmarks/kdebug_repair_benchmark/scripts/run_matrix.sh \
  --suite-root /home/host/kverif_runs/suite \
  --bench-root /home/host/kverif/benchmarks/kdebug_repair_benchmark \
  --models qwen3.6-35b \
  --timeout 3600 \
  --evidence-mode collect
```

The lower-level single-trial command below is for runner development only. It
assumes collection and `validate-suite` have already succeeded:

```bash
export MAAS_BASE_URL=https://maas-coding-api.cn-huabei-1.xf-yun.com/v2
export MAAS_API_KEY=<provided key>
export RESULTS_CSV=/home/host/kverif_runs/suite/results.csv
export KDEBUG_PYTHON=/usr/local/bin/python3.8
export RUN_AGENT_CMD='"$KDEBUG_PYTHON" /home/host/kverif/benchmarks/kdebug_repair_benchmark/scripts/api_model_runner.py --model "$MODEL" --repair-dir "$REPAIR_DIR" --group "$GROUP" --timeout "$TIMEOUT_SEC" --results-csv "$RESULTS_CSV"'

bash /home/host/kverif/benchmarks/kdebug_repair_benchmark/scripts/run_repair_trial.sh \
  --suite xiangshan_repair_benchmark_v2 \
  --case-dir /home/host/kverif_runs/suite/case_001 \
  --repair-root /home/host/kverif_runs/suite/repair \
  --model glm-4.7 \
  --group without_kdebug \
  --timeout 3600
```

After trials finish:

```bash
/usr/local/bin/python3.8 /home/host/kverif/benchmarks/kdebug_repair_benchmark/scripts/collect_results.py \
  --repair-root /home/host/kverif_runs/suite/repair \
  --out /home/host/kverif_runs/suite/results.csv

/bin/python3 /home/host/kverif/benchmarks/kdebug_repair_benchmark/scripts/capture_terminal_screenshots.py \
  --results /home/host/kverif_runs/suite/results.csv \
  --log-root /home/host/kverif_runs/suite/repair \
  --out-dir /home/host/kverif_runs/suite/screenshots \
  --overwrite \
  --require-logs

/bin/python3 /home/host/kverif/benchmarks/kdebug_repair_benchmark/scripts/generate_word_reports.py \
  --results /home/host/kverif_runs/suite/results.csv \
  --screenshots /home/host/kverif_runs/suite/screenshots \
  --out-dir /home/host/kverif_runs/suite/docx_out
```
